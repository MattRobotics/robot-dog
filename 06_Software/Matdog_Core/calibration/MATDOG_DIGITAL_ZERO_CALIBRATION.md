# MATDOG — 12-Servo ST3215 Digital-Zero Calibration

## Purpose

`matdog_digital_zero_calibration.py` is the canonical MATDOG utility for calibrating the digital zero of all 12 ST3215 leg servos in one controlled batch.

The operator first places every joint manually in the validated mechanical `q = 0` pose with torque disabled. The utility then calculates the signed ST3215 Position Offset required to represent that pose as encoder position `2048` for every joint.

The complete workflow is:

```text
manual mechanical q = 0
    -> read-only preflight and immutable plan
    -> one authorized 12-servo EEPROM batch
    -> Station restart
    -> final read-only EEPROM verification
```

## Scope boundary

Digital-zero calibration and mechanical end-stop calibration are separate
operations.

This procedure maps the validated mechanical `q = 0` pose to displayed encoder
position `2048`. It does not search for physical end stops, measure contact
limits, set safe travel limits or authorize stand motion.

The next calibration phase is defined in:

```text
06_Software/Matdog_Core/calibration/
MATDOG_MECHANICAL_ENDSTOP_CALIBRATION_PLAN.md
```

## Safety contract

The utility is MATDOG-specific and enforces the following rules:

- NormaCore Station remains the sole serial owner.
- Exactly the 12 canonical MATDOG servo IDs must be present.
- Torque must be disabled on all 12 servos.
- No `GoalPosition` command is generated.
- No torque-enable command is generated.
- The preflight pose must be stable and within the configured tolerance.
- Every EEPROM write is serialized per servo.
- Every servo is unlocked only for its own offset operation.
- EEPROM relock is executed in a `finally` block.
- Live `PresentPosition` must confirm the expected digital recenter.
- Transient incomplete inference frames are ignored until a complete frame is available.
- A valid final calibration requires a Station restart followed by EEPROM read-back of all 12 servos.
- The client connection and asynchronous tasks are closed automatically before process exit.

The generic NormaCore automatic-calibration commands are not part of this procedure.

## Canonical servo order

```text
LF: M13 hip, M12 upper, M11 lower
RF: M23 hip, M22 upper, M21 lower
RH: M33 hip, M32 upper, M31 lower
LH: M43 hip, M42 upper, M41 lower
```

## 1. Mechanical preparation

1. Suspend or safely support MATDOG.
2. Keep all servo torques disabled.
3. Manually place all 12 joints in the validated mechanical `q = 0` pose.
4. Do not move the robot after starting the preflight.
5. Start NormaCore Station and confirm that it is the sole owner of the ST3215 serial adapter.

## 2. Read-only preflight

```bash
cd ~/MATDOG/github/robot-dog

python3   06_Software/Matdog_Core/calibration/matdog_digital_zero_calibration.py   preflight
```

Optional controls:

```text
--output-dir
--samples
--stability-spread
--q0-tolerance
```

The preflight:

- captures multiple complete 12-servo samples;
- verifies torque OFF and EEPROM lock state;
- verifies pose stability;
- records the current physical encoder values;
- calculates all 12 signed Position Offsets;
- predicts the resulting displayed value near `2048`;
- writes a pre-recenter backup, mechanical-q0 snapshot and immutable digital-zero plan;
- writes SHA-256 sidecars for the generated records;
- performs no EEPROM write and sends no motor command.

## 3. Execute the complete 12-servo batch

Use the plan generated by the preflight:

```bash
python3   06_Software/Matdog_Core/calibration/matdog_digital_zero_calibration.py   execute   --plan <PATH_TO_DIGITAL_ZERO_PLAN.json>   --confirm "WRITE ALL 12 DIGITAL ZERO OFFSETS"
```

For each servo, the program performs:

```text
global torque-OFF and pose gate
    -> verify current offset and EEPROM lock
    -> unlock EEPROM
    -> RegWrite signed Position Offset at 0x1F
    -> Action
    -> verify live PresentPosition
    -> relock EEPROM
    -> verify lock = 1 and torque = 0
```

The operation is serialized internally but requires only one operator command and one exact confirmation string.

The execution audit is updated after every completed servo. The program exits automatically after completing or aborting the batch.

## 4. Restart Station

After successful execution, stop and restart Station. This forces a complete EEPROM read and clears any previously cached register values.

Do not reposition the joints between execution and final verification.

## 5. Final read-only verification

```bash
python3   06_Software/Matdog_Core/calibration/matdog_digital_zero_calibration.py   verify   --plan <PATH_TO_DIGITAL_ZERO_PLAN.json>
```

Final acceptance requires, for all 12 servos:

- EEPROM Position Offset equal to the planned signed value;
- displayed position within tolerance of `2048`;
- physical raw encoder position within tolerance of the mechanical-q0 capture;
- EEPROM lock equal to `1`;
- torque equal to `0`.

The verification mode performs no EEPROM write and sends no motor command.
